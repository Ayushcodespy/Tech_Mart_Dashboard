# Project Documentation

This folder contains detailed SRS-style documentation for both customer-facing and admin-facing parts of the grocery e-commerce platform.

## Files

- `user-app-srs.md`
  - Customer mobile/web app requirements and flows
- `admin-panel-srs.md`
  - Admin/staff dashboard requirements and operational flows

## Scope

The documentation reflects the current project direction:

- Backend: FastAPI
- Database: PostgreSQL
- ORM: SQLAlchemy
- Authentication: JWT
- Customer frontend: Flutter
- Admin frontend: React + Vite
- Storage: Local file storage, designed for future cloud migration

## Notes

- These documents are written as implementation-oriented SRS documents.
- They can be used for development planning, QA, handoff, onboarding, and future scaling decisions.
